# Closeout contract

- Keep this capsule single-purpose; route another lane through its own owner and capsule.
- Keep generated, private, and heavy files classified.
- Push useful source work or create a durable owner receipt before local cleanup.
- Never delete personal/private data without its explicit custody gate.
- Record changed paths, the once-per-head predicate result, exact remote custody, owner, merge
  condition, and any owner-routed residual.
- Do not call GitHub blocked because a hosted job executed zero steps.
- Preserve the admitted runway across successor sessions. A genuinely new runway requires a new
  successor contract; never reset this capsule's deadline.
- Reach two read-only unchanged closeout passes: no dirty/unpushed session worktree, no uncapsuled
  local branch, no old root falsely active, and every residual represented by a PR or owner receipt.
- At a session boundary, leave the next capsule index and one launch command.
