MONETA

Sell Pro licences for your software with no payment processor anywhere in the path — a self-hosted Bitcoin licence mint you run in one container.

Get the source See a live mint

What it does

buyer ──pays sats──▶ your Bitcoin address │ (you hold the keys; MONETA never touches funds) buyer ──GET /order──▶ MONETA ──reads chain──▶ mempool.space (read-only, no API key) │ └─signs licence with MONETA's key─▶ buyer buyer pastes licence ─▶ your product verifies it OFFLINE against the public key

A buyer pays Bitcoin straight to an address you control. MONETA watches a keyless public block explorer (mempool.space/Esplora — read-only, no API key, swappable for your own self-hosted instance) and, on confirmed payment, signs your product's own ECDSA-P256 Pro licence — the same offline-verifiable token your licence core already checks. No Stripe, no Lemon Squeezy, no Ko-fi, no account, no KYC, nothing a third party can freeze or revoke.

What it never does

MONETA is read-only against the chain. It holds no spending keys, signs no transactions, and cannot move your funds — it only asks the explorer whether N sats landed at your address. The only key it holds is the licence-signing key, a code-signing key auto-generated on first boot.

No sale is ever lost

Unconfigured, it still takes orders: before a receive address is set, /checkout pools each buyer as a reserved order and /health reports the waiting count. The instant you set MINT_BTC_ADDRESS, every pooled order opens into a payable one and mints on confirmation. The valve opens; demand was never turned away.

Run it ($0 deploy)

docker build -t moneta . docker run -p 8787:8787 --env-file .env -v "$PWD/.data:/app/.data" moneta # or: npm ci && npm start → http://localhost:8787

One env value is sovereign: MINT_BTC_ADDRESS, from any wallet you control. Everything else has working defaults.

The endpoints

GET /Self-contained buyer checkout page — BTC address, unique sat amount, BIP21 link, polls to confirmation, returns the buyer to your product with ?ce_license_key=…
GET /pubkeyThe public JWK you embed in your product build
POST /checkoutNew order (201 payable, or 202 reserved while the valve is closed)
GET /order/:idPoll; mints and returns the licence on confirmed payment
GET /healthLiveness, configured flag, pooled-demand count

Wire it to your product in three steps

  1. npm run keygen — prints the public JWK (the private half stays a deploy secret).
  2. Set the public JWK in your product's build so it verifies licences offline.
  3. Point your “Buy Pro” button at the mint's checkout; the buyer returns with the licence in the URL and your product captures it.

First consumer in production: the ChatGPT Exporter — its Buy Pro button opens a live MONETA mint, and purchased keys unlock Pro with zero network calls.

Juno Moneta: the Roman mint, the temple where coin was struck — the literal root of “money” and “mint.” Receiving money to a self-custodied wallet is the only rail a private party can run end-to-end with no account, no KYC, and no middleman. MONETA supplies the missing half: minting offline-verifiable licences on confirmed payment, instead of renting a processor.
