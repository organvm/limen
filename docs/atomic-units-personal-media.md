# Atomic units for personal media — iCloud Drive + Photos into the remix/transmutation system

> Design (strand D, 2026-06-22). Per his words: *"iCloud gate is a you[=system] gate — we need to
> design how to process and store it as atomic units for remix and transmutation in the system —
> photos as well."* Storage reclamation (strand C) is the byproduct; the goal is to make his personal
> media first-class **atoms** in the same capture→converge→mesh engine that already distills his words.

## The insight

The system already has the atom abstraction and the engine — it just doesn't feed personal media into
them yet:

- `cli/src/limen/converge.py` — `Shot(id, text, source)` is THE atomic unit; `converge()` distills N
  shots → ONE; `MeshRanker`/`MeshGapFinder` put atoms in an influence/dead-zone graph.
- `scripts/corpus-converge.py` — the live organ: atomize → bucket by face → distill → THE ONE (now
  closing the loop keylessly via `ClaudeCliSynthesizer`, strand A).
- `scripts/capture.sh` — pulls content OFF disk into the canonical distributed graph.

Personal docs and photos become `Shot`s. Then **remix** = `converge()` across them; **transmutation** =
distill/link them to his words, generate timelines/albums/narratives; **storage** falls out for free —
once an item is atomized + stored canonically + offsite, the local original is safely evictable.

## The pipeline (one new organ, reusing the engine)

`scripts/media-atomize.py` (mirrors `library-preserve.py`'s safety: dry-run default, copy→verify,
never-delete-source, fail-open). Four stages:

1. **INGEST** — walk two sources, emit raw items (path, mtime, type):
   - iCloud Drive `…/CloudDocs` (already copied to Archive4T by strand C, so atomize from the durable
     copy — no re-download): PDFs, md/txt, notes, docs.
   - Photos: enumerate via the Photos library (`Photos.sqlite`) or the on-disk `originals/` — get
     image path + EXIF (time, GPS, camera) WITHOUT importing the library.

2. **ATOMIZE** — turn each raw item into one or more `Shot`s:
   - Doc → extract text (pdftotext / plain read) → chunk into section-level shots
     `Shot(id=hash, text=section, source=path)`.
   - Photo → a metadata+caption shot: `text` = a vision caption + detected objects/scene (generated by
     the **keyless `claude -p` with the image**, the same subscription-authed path strand A proved —
     no API key, no spend beyond CLI budget), plus EXIF as structured fields; `source` = image path;
     the image bytes stay in Archive4T/iCloud (pointer only, never inlined).

3. **STORE** — write atoms into the canonical corpus store the converge engine already reads
   (content-addressed, dedup by hash), so `capture.sh` pushes them off-disk into the universal context.
   Idempotent: re-runs only atomize new/changed items.

4. **REMIX / TRANSMUTE** — the EXISTING engine now operates over the enriched atom set:
   - cross-modal `converge()` — "distill my year": photos + messages + journal docs + his words at one
     idea → ONE narrative/album/timeline.
   - mesh influence/dead-zones surface the connective tissue (a place that recurs across photos +
     messages + notes) and the gaps (a trip with photos but no written reflection → a next shot).

5. **RECLAIM (byproduct)** — once an item is atomized + stored + offsite-verified, evict the local
   original (iCloud: `brctl evict`, already live in `library-preserve.py`; Photos: optimize/evict the
   originals whose atoms+bytes are durable). Reversible; the atom + the Archive4T/cloud bytes remain.

## Safety / envelope (same hard rules)

Never auto-delete personal data; copy→verify→never-delete-source; evict (reversible) only after the atom
AND the bytes are durably stored. Vision captioning is keyless (no spend beyond CLI). Photos library is
read via its DB/originals, never mutated. Dry-run first; the daemon runs it slow (a media voice near
`C_BACKUP`) and fail-open.

## Build slices (bounded, in order)

1. **Docs slice** (cheapest, highest signal): atomize the already-on-Archive4T CloudDocs text/PDF/md →
   shots → store → let `corpus-converge` remix them with his words. No new infra; reuses `Shot`+converge.
2. **Photos metadata slice**: EXIF + path atoms (no vision yet) → a queryable photo-atom layer + the
   time/place graph. Cheap, no model calls.
3. **Photos caption slice**: keyless `claude -p` vision captions, bounded per beat (like corpus-converge's
   `--limit`), enriching photo atoms into full remixable shots.
4. **Cross-modal convergence**: a face/idea that draws from docs + photos + messages + words → THE ONE.

Each slice is independently shippable and reclaims its source as it completes.
