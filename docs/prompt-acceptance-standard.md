# Prompt Acceptance Standard

Recurring prompts are not accepted just because they were indexed. A serious prompt packet is
accepted only when it closes the full frame below without leaking raw prompt text or private
session evidence into tracked files.

## Required Frame

Every acceptance receipt must answer, in order:

1. `prompt_receipt` - What did we prompt?
2. `evolved_intent` - What newer form of the idea got addressed, and what older lineage still
   matters?
3. `owner_outcome` - What remains owner-recorded, closed, parked, or still needing a route?
4. `august_runway_impact` - Does this help the Aug-1 operating checkpoint and the late-August
   unemployment runway?
5. `outward_reciprocity` - What did we observe, absorb, stage, gate, deliver, or mark not
   applicable?

The ordering rule is: older ideas carry lineage weight, but a newer evolved form wins when the idea
has moved through new lenses, portals, mirrors, or reflections.

## Public Boundary

Tracked docs may include packet ids, batch ids, family names, status values, counts, and
public-safe summaries. They must not include raw prompt bodies, secrets, personal contact data,
customer data, private crash bodies, private session paths, full prompt hashes, or session ids.

Ignored private indexes may keep full hash/session evidence under
`.limen-private/session-corpus/lifecycle/`.

## Reciprocity

Outward reciprocity is a full loop, not a reduction:

- `observed` - outside work or an external failure mode was noticed and recorded.
- `absorbed` - the lesson changed local practice.
- `staged` - a useful contribution or reflection is drafted locally.
- `gated` - a human decision is required before identity-bearing outbound action.
- `delivered` - the contribution was sent or published with explicit human approval.
- `not_applicable` - no outward loop is required for this packet.

No GitHub issue, PR, email, bug report, public post, or other identity-bearing outbound action is
sent automatically from this layer. Staged or gated receipts stop at the human gate.

## August Runway

The operational checkpoint remains `2026-08-01`. The late-August unemployment runway is the hard
premise that makes the checkpoint matter. Acceptance work should shorten the path to received cash,
booked calls, clean owner records, and fewer repeated prompt loops; it should not extend internal
tooling for its own sake.
