## Truth contract

This is an epoch closeout, not a claim that the whole estate is finished. An epoch leaf is closed
when it has one of the lifecycle receipts accepted by `AGENTS.md`: a verified merge, an open owner PR
with a predicate and merge condition, a pushed owner task/plan, a preservation receipt with a next
action, or a named external gate. Historical debt remains visible; it is not coerced into `done`.

The snapshot excludes raw prompt bodies, active Claude worktree diffs, and unplugged external
volumes. GitHub is authoritative for remote custody. Counts that were observed on a moving surface
are labeled as snapshots rather than eternal facts.

## Verdict

Real work shipped, but intake and rework grew faster than terminal closure. The system produced a
high volume of durable GitHub receipts while spending too much motion on reopen/heal cycles and too
little on value underwriting. The correct next move is a fresh continuation capsule driven by live
predicates, not a longer mega-session and not a fictional declaration of global Omega.

## Insight

1. **Throughput is real; convergence is weak.** An 88.08% terminal PR yield coexists with a 90.6%
   reopen-to-dispatch event ratio. The fleet can land code, but it revisits too many obligations.
2. **The denominator is winning.** Terminal stock gained 179 tasks while nonterminal stock gained
   106; closure percentage still fell because 285 tasks entered the board.
3. **Evidence is not completion.** Prompt evidence coverage reached 94.5%, while strict terminal
   prompt authority remains unsealed. Any combined progress number would be materially misleading.
4. **Provider capacity is not the main bottleneck.** The latest receipt showed substantial Claude,
   Jules, Gemini, and Agy headroom; ownership, predicates, conflicts, and host/resource gates were the
   limiting factors. OpenCode alone was near throttle.
5. **A mega-session becomes coordination debt.** Once handoff, board, main, and active worktrees move
   underneath the session, continuing the same context increases rework. A continuation capsule is a
   control-plane primitive, not clerical ceremony.

## Evolve

### Continuation capsule standard

Every closeout and every new autonomous initiative must emit one self-contained continuation
capsule:

1. an isolated worktree and single-purpose branch;
2. a README containing the objective, authorities, prohibitions, evidence links, and first probes;
3. one copy/paste launch command;
4. a live environment snapshot generated at launch, not frozen model/provider assumptions;
5. executable completion predicates and ownership rules, never a predeclared verdict;
6. a requirement that every discovered leaf ends merged, owner-PR'd, preserved, or externally gated;
7. a relay back into the next capsule when the context, value, resource, or provider gate says to
   switch sessions.

The runtime derives the ending from current facts: exact remote head, CI state, task contracts,
handoff age, provider headroom, mounted substrates, host pressure, lifecycle custody, and strict
Omega/no-task predicates. Environment figures select the lane and decide whether work may continue;
they never get edited to manufacture green.

This converges the already-shipped `scripts/start-worktree-session.sh`, `.limen-workstream/README.md`,
`scripts/handoff-relay.py`, and `scripts/continuation-beat.py` surfaces instead of creating another
parallel launcher.

### Work-loan and progress contract

- Admit work only with an owner, executable predicate, durable receipt target, expected value, cost
  ceiling, horizon, dependencies, and stop/switch conditions.
- Show separate progress bars for discovery, underwriting, execution, remote custody, predicate
  verification, and terminal closure.
- Derive providers from live catalogs/telemetry. Never encode a fixed model table or treat a local
  concurrency ceiling as a global fleet ceiling.
- Freeze repeated HEAL generation by live `(repo, failing-check)` ownership groups and require every
  heal receipt to carry an outcome.

## Next-session contract

The closeout must leave a generated worktree README and one launch command here. The prompt may name
the current owner receipts, but it must begin by re-probing remote main, CI, board stability, handoff,
provider headroom, mounts, host pressure, and active sessions. It may define predicates for closure;
it may not assert in advance which lane is next or that Omega is reachable.
