# Evolve

Every closeout and autonomous initiative emits one continuation capsule:

1. an isolated worktree/branch when repository-backed, otherwise an owner-native workspace/receipt;
2. a thin README prompt index linking cohesive intent, runtime, closeout, and evidence modules;
3. one copy/paste launch command;
4. a live environment snapshot generated at launch, not frozen provider assumptions;
5. executable completion predicates and ownership rules, never a predeclared verdict;
6. terminal custody for every discovered leaf;
7. a successor capsule when live context, value, resource, or provider gates require a boundary.

The runtime derives the ending from exact remote/CI state, task contracts, handoff age, provider
headroom, mounted substrates, host pressure, lifecycle custody, and scoped predicates. Evidence and
thresholds are never edited to manufacture green.

## Modular artifact and verification contract

- One module owns one responsibility, owner, and independently runnable contract.
- README files are indexes, not concatenated prompts, reports, logs, or transcripts.
- An aggregate verifier may fan in bounded shards; it may not repeat their work, hide an unbounded
  loop, or become the only way to run a component predicate.
- Each shard emits a small receipt, has a configured timeout/retry policy, and keeps logs bounded.
- Split at semantic ownership seams, not an arbitrary line count. If unrelated changes repeatedly
  touch one file, the file has already disclosed the missing seam.

The executable owner contract for the current duplicated CI path is
`docs/plans/ci-bounded-shards.md`.

## Work-loan and progress contract

- Admit work only with an owner, predicate, receipt target, expected value, cost ceiling, horizon,
  dependencies, and stop/switch conditions.
- Show separate progress for discovery, underwriting, execution, custody, predicate verification,
  and terminal closure.
- Derive providers from live catalogs/telemetry; never encode a fixed model table.
- Freeze repeated HEAL generation by live `(repo, failing-check)` groups and require outcomes.
