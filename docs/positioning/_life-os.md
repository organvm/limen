# The Life Operating System

**The same architecture that runs a fleet of AI agents runs an entire human life. That's the proof the method generalizes.**

Most "AI assistant" demos stop at a to-do list. The real test of an autonomic system is whether it can hold a whole life — the boring, high-stakes, easy-to-drop parts — and keep every lane running without being asked. This one does, on exactly the architecture the agent fleet runs on: **source of truth → metabolize → route → dispatch, with only the irreversible steps gated.**

## Five lanes, one engine

The lanes aren't repositories — they're life-domains: **inbox, money, health and body, communications, and ventures.** The same cycle runs across all five: explore → plan → build → verify → heal → learn → repeat.

## The authority model — the part that makes it trustworthy

| Bucket | What the system may do | Gate |
|---|---|---|
| **Back-office / reversible** | Triage the inbox, plan labels, draft replies, surface bills and deadlines, build runbooks, structure the day. | **None. Autonomic.** |
| **Money** | **Propose-only. Never moves money.** Surfaces the obligation, stages the one action, marks it yours. | **Hard gate.** |
| **Outward sends** | Composes and stages; never sends on its own. | **Soft gate — intentional friction.** |

The default posture is the whole design: **mark every outward or irreversible atom as yours, stage it fully prepped as one tap, and keep every other lane running in the meantime.**

## Everything that hangs on the human lives in one place

No obligation lives in a chat. Every human-gated lever and every credential lives in a durable, individually-closeable registry — surfaced once, never nagged, each with the cheapest path to resolution attached. Close the item to pull the lever. The registry regenerates itself continuously; there's no manual sync.

This is the root-to-leaf discipline: when one action unblocks a chain of downstream locks, the system finds that root action and stages *it* — not the ten symptoms.

## The ethos: never "can't"

A blocker is not a stopping point — it's the start of work. Log it, contextualize it, decompose it to its irreducible atom, explore the free path first, tee up the single click, and keep every other lane moving. Nothing is discarded; it gets evolved until it works.
