# Objective

Move the implementation PR from protocol scaffold to a production-safe, live peer mesh, then execute
the whole-PR campaign only as far as current authority, provider headroom, and exact-head evidence
allow.

The finish line is executable:

- the implementation and organization-profile PRs are merged at exact green heads;
- remote conduct authentication binds a credential to a server-derived principal, session,
  capability set, and root authority;
- lease capability is delivered only to the selected executor and is recoverable after response
  loss without disclosure to a conductor;
- Python and Worker kernels make the same decision for shared parity fixtures;
- all live child launchers reserve through the broker and inject actual lease lineage;
- the exact review-gate owner proves checks, reviews, threads, and readiness beyond API page limits;
- a complete native five-by-five initiator/executor canary matrix returns schema-valid receipts;
- every current PR head has a live disposition and durable owner/receipt, all eligible work
  terminates, and an immediately repeated complete pass creates no new work;
- direct-writer, expired-lease, duplicate-work, unharvested-result, and unowned-resource counts are
  zero.

Until all of those pass, the correct boundary is `continue`, `switch`, or `wait_relay`, never
`settled`.
