## The Map (Micro Instance)

This is the artifact required by `organ-ladder.json` for maturity stage 30% ("building"): "a personal
balance-sheet + cashflow + payrail disbursement map run on his real numbers". Real financial data goes here
(locally stored or securely referenced via `the-invisible-ledger`).

### 1. Balance Sheet (Net Position)
*Tracks: Assets vs. Liabilities across all controlled entities.*

*   **Assets:**
    *   Operating Cash (Personal)
    *   Treasury / Institutional Cash
    *   Equities / Investments
    *   Illiquid Assets / IP
*   **Liabilities:**
    *   Current Payables (Cloud, Subscriptions, APIs)
    *   Taxes Owed (Estimated)
*   **Net Worth (Standing):** [Calculated from Ledger]

### 2. Cashflow (Runway & Burn)
*Tracks: The heartbeat of the system. Money in vs. Money out.*

*   **Inflows (Revenue):**
    *   Stripe / Product Revenue (Mirror Mirror, Invisible Ledger)
    *   Consulting / Retainers
*   **Outflows (Burn):**
    *   API / LLM Token Spend (Anthropic, OpenAI)
    *   Infrastructure (Cloudflare, Firebase, GHCR)
    *   Personal Burn (Living Expenses)
*   **Runway:** [Calculated from Cash on Hand / Monthly Burn]

### 3. Payrail Disbursement Map
*Tracks: How a dollar is routed once it enters the system (the "invisible ledger" rule engine).*

When product revenue arrives (e.g., via Stripe):
1.  **Tax Withholding (30%):** Automatically earmarked for tax liabilities.
2.  **Institutional Treasury (40%):** Reinvested into API compute, infrastructure, and contractor runway.
3.  **Personal Draw (30%):** Disbursed to personal accounts for living expenses and private investments.

*This rule engine is what turns `the-invisible-ledger` from a retrospective CPA tool into a proactive family office.*
