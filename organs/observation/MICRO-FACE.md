# OBSERVATORY — MICRO FACE (Anthony's own instance)

The estate's activation program. The diagnosis it was built to act on, in the spec's own words:

> **You do not have a production deficit. You have an activation, compression, concentration, and
> distribution deficit.**

The `organvm/portfolio` surface is the evidence: a technically serious system (19 case studies, live
site, ~709 commits) showing **0 stars / 0 forks** — production that has not converted to visible
external adoption. Compounding it, a public-legibility problem: the homepage, README, and portfolio
have reported **105 / 116 / 171** repositories, which reads externally as claim drift.

## What the micro instance does

**Internal legibility (reconcile our own claims).** The 105/116/171 split is *partly intentional* —
`face-ownership.json` declares `total_repos`=171 (live gh-api) and `curated_repos`=149 (curated) as
both true; the failure is faces binding the wrong variable and a **severed CI pipe** (the
`build_vars → system-vars.json` step dropped from `metrics-refresh.yml` on 2026-05-22, freezing the bio
at 148). OBSERVATORY supplies the missing *effector* for the loop whose sensor (VVLTVS) already exists:
it revives the pipe and makes each face project from its register, proposing the cross-repo PRs — it
does **not** become a fourth stamper.

**External legibility (study the field).** Each morning it collects the day's GitHub winners
(PageAgent, OfficeCLI, `mattpocock/skills`, and the like), selects matched controls, extracts each
README's communication surface, and scores which winning mechanisms are *transferable and cheap* for us
— a one-sentence promise, a one-line install, a 15-second visual proof — versus the inherited advantages
we cannot copy (Apple's brand, a 60K-person newsletter).

**One experiment.** It picks the hero repo (`value-repos.json` rank 1) and proposes one reversible
change — replace an abstract hero with an executable demo, cut install from eight steps to two,
reconcile a contradictory claim — with a measurement contract, and tracks visits → proof → trial → use
→ return → inquiry.

## The first campaign (30 days)

Not 400 projects at once. Inventory the public estate → identify the ~25 closest to a functioning
external product → rank by activation potential → select **one hero** → let three supporting repos serve
as evidence/SDK/docs → run one activation experiment per day on the hero → measure the funnel. The first
project need not represent everything; it needs to provide the first clean entrance into everything else.

> **The strategic reversal:** one successful entrance first; the institution becomes the explanation later.

## The one human act

Everything above is the organ's. The only thing that stays Anthony's is **approving** the day's proposed
experiment when it touches a public surface — filed as a `his-hand-levers.json` lever, read on his own
cadence, never recited back to him.
