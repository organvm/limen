# BIFRONS — Kernel (the five primitives)

**BIFRONS** (Janus, two-faced) is the star ↔ contribution portal: every repo we star is both
*absorbed* (inbound) and *contributed-to* (outbound), one `exchange_id` threading the whole
traversal. The loop mechanics live in three merged repos (alchemia intake, ontologia identity,
organvm-engine compile/decide/execute); this organ is the **heartbeat face** that keeps the loop
alive and renders its proof surface.

| Primitive | BIFRONS |
|---|---|
| **Member** | a starred external repo (its dossier + its exchange) |
| **Mandate** | absorb its wiring into ORGANVM *and* prepare a contribution back |
| **Standing** | the exchange lifecycle: `STARRED → INDEXED → DOSSIERED → MAPPED → {ABSORPTION → INTERNAL_PR_OPEN} / {CONTRIBUTION → … → HUMAN_APPROVED → UPSTREAM_SUBMITTED} → BACKFLOW_COMPLETE` |
| **Standard** | one `exchange_id` threads star → backflow; two independent scores (absorption vs contribution) never collapsed; **A2 prepare-never-submit** |
| **Governance** | human-gated at exactly one edge — opening the upstream PR — which rides the existing outbound-send valve (SPECVLVM), never a BIFRONS-owned lever. Everything prior is autonomous. |

## The autopoietic contract

- **Reaches into the past** — the beat crawls its own state estate (`~/.organvm/bifrons/`) and the
  portal store, regenerating understanding from history, never a hand-fed input.
- **Exists in the present** — wired to the `bifrons-portal` heartbeat sensor; ships LIVE, runs
  unbidden on cadence (read-only absorb + internal prepare — no send, no spend).
- **Builds the future** — owns zero `his-hand` levers; prepared contributions pool at the gate and
  self-surface as a count. It asks less of the operator over time.

The arbiter of "done" is `scripts/avtopoiesis.py --strict` counting `bifrons` a live door.
