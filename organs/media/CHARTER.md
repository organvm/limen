# Carrier-Wave Media — CHARTER (the cross-platform media empire)

> **Boundary:** this is AI-run media infrastructure that augments an editor-in-chief / public figure.
> It does **not** speak externally autonomously. The human directs, approves, and owns every broadcast.
> See [KERNEL.md](KERNEL.md) for the full map and guardrails.

## What it rivals

A global PR team or media empire does this for an ultra-rich figure: it ensures a consistent voice, manages cross-platform formats, tracks engagement, and maintains a regular publication heartbeat.
Carrier-Wave Media supplies that with AI roles and explicit human authority.

| Role | What it does | Human check |
|---|---|---|
| **Editor-in-Chief (the human)** | vision, message selection, voice, final publish approvals | — (final authority) |
| **Drafter** | turns system events into raw drafts | human rewrites and owns language |
| **Social Distributor** | translates the core essay/message into platform-native idioms (Mastodon, Bluesky, LinkedIn, Discord) | human reviews and approves translations |
| **Publisher** | stages the approved drafts in the distribution pipeline (`kerygma-pipeline`) | human clicks publish/send |
| **Analytics Collector** | tracks engagement, telemetry, and reach across channels | human reviews metrics reports |

The point is not labor substitution. The point is institutional continuity: the message is distributed coherently under one human intellect without the friction of manual formatting and cross-posting.

## The workflows it runs

1. **Intake → Draft.** System events (e.g., essays generated by `essay-pipeline`, GitHub releases) are ingested.
2. **Translate → Format.** The Drafter creates platform-specific text using `announcement-templates`.
3. **Stage → Preview.** The Social Distributor generates a preview artifact (the "staged release").
4. **Publish → Broadcast.** ONLY AFTER the human's explicit gate, the Publisher dispatches via `kerygma-pipeline`.
5. **Measure → Report.** Analytics Collector records the delivery log and subsequent engagement.

## Inputs / outputs

- **Inputs:** raw system events, generated essays (from `essay-pipeline`), release notes, and the human's explicit approval.
- **Outputs:** Drafted social posts, staged pipeline runs, delivery logs, and analytics reports.

## Current micro proof (active instance)

The micro instance is the **ORGANVM public face**, integrating `essay-pipeline` with `kerygma-pipeline`.
The primary workflow is taking the automated Sprint Narratives and system essays and broadcasting them across POSSE channels (Ghost as primary, syndicated to Mastodon, Discord, Bluesky, and LinkedIn).

## Boundaries (always)

- No autonomous publishing. Every dispatch requires a human click.
- Platform constraints (character limits, media requirements) are handled by the organ, but the message itself is owned by the human.
- Do not invent facts or metrics.
- Every drafted post is advisory until published.
