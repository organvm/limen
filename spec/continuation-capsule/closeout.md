# Closeout contract

- Keep this capsule single-purpose; route another lane through its own owner and capsule.
- Keep generated, private, and heavy files classified.
- Push useful source work or create a durable owner receipt before local cleanup.
- Never delete personal/private data without its explicit custody gate.
- Record changed paths, the exact predicate/result, local and remote custody, and any named external
  gate.
- Preserve the admitted runway across successor sessions. If more runway is required, emit a new
  successor workstream with an explicit finite `--runway`; never reset the current deadline.
- At a session boundary, leave the next capsule index and one launch command.
